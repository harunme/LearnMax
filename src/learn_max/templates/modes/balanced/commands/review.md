## 上下文

- 学习目录：!`ls -d .learning 2>/dev/null`
- 当前主题：!`ls .learning/ 2>/dev/null`

**注意：** 如果 `.learning/` 不存在，提示用户运行 `/learn [主题]`。检查主题文件夹（忽略 `scripts/`）。

## 你的任务

进行间隔重复复习，对抗遗忘并强化学习。

**如果没有 `.learning/`：**

- 提示："没有进行中的学习。使用 `/learn [主题]` 开始！"

**如果有待复习：**

对于复习列表中的每个概念：

1. 呈现："让我们复习：[概念名称]"
2. 提示教学（轮换）：
   - "用你自己的话解释 [概念]"
   - "你会怎么向初学者教 [概念]？"
   - "[概念] 背后的核心思想是什么？"
3. 听取用户的解释
4. 评估：
   - 清晰且准确 → 表扬，标记已复习
   - 部分正确 → 问澄清问题，引导填补差距
   - 不正确 → 温和纠正，提供提示
5. 标记已复习：`python3 .learning/scripts/review_scheduler.py review <主题-slug> "[概念]"`

**所有复习后：**

- 庆祝："太棒了！复习了 N 个概念！🎉"
- 显示下次复习日期
- 使用 `AskUserQuestion` 询问下一步：

```json
{
  "question": "你接下来想做什么？",
  "header": "下一步",
  "multiSelect": false,
  "options": [
    {
      "label": "学习新的",
      "description": "继续下一个大纲项目"
    },
    {
      "label": "练习",
      "description": "进行动手练习"
    },
    {
      "label": "休息一下",
      "description": "稍后再来"
    }
  ]
}
```

**如果没有待复习：**

- 提示："今天没有待复习！下次：[日期]"
- 建议继续新材料

**处理遗忘的概念：**

- 不要立即给出答案
- 提供提示："这与 [上下文] 相关..."
- 如果仍然卡住：简要复习，重新安排到明天
- 重新安排：`python3 .learning/scripts/review_scheduler.py add <主题-slug> "[概念]"`

**关键原则：** 主动回忆（用户从记忆中重建），而非被动识别
