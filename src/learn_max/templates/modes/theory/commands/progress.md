## Context

- 学习目录: !`ls -d .learning 2>/dev/null`
- 当前主题: !`ls .learning/ 2>/dev/null`

**注意:** 如果 `.learning/` 不存在，请告知用户运行 `/learn [topic]`。检查主题文件夹（忽略 `scripts/`）。

**注意:** 获取主题 slug 后，读取必要的文件：

- `.learning/<topic-slug>/metadata.json`
- `.learning/<topic-slug>/progress.md`（最后 30 行）
- `.learning/<topic-slug>/syllabus.md`
- `.learning/<topic-slug>/review_schedule.json`

## Your Task

生成一份鼓励性的进度报告，展示学习历程和下一步行动。

**如果没有 `.learning/`：**

- 告知用户："还没有进行中的学习。使用 `/learn [topic]` 开始吧！"

**计算指标：**

- 已完成的学习次数（从 metadata 获取）
- 开始后的天数（从 created_at 计算）
- 已学概念数量（从 progress.md 统计）
- 大纲进度（统计 `[x]` vs 总计 `[ ]`）
- 已完成的复习次数（从 review_schedule.json 获取）
- 当前阶段（从大纲位置确定）

**展示报告：**

```markdown
📊 进度报告：[主题名称]

🎯 概览
学习次数：[N] | 天数：[N] | 阶段：[X] - [名称]
大纲：[X]%（[M]/[总数] 项）

📚 已学概念
✓ [概念 1]
✓ [概念 2]
...（共 [N] 个）

🎉 近期成就

- [成就 1]
- [成就 2]

📅 复习统计
已完成：[N] | 已安排：[N]
下次复习：[日期]（[N] 个概念）

🎯 下一步重点

1. [下一个未完成项]
2. [下一个未完成项]

💡 洞察
[根据数据个性化生成]
```

**庆祝里程碑：**

- 5 次学习："🎉 正在建立动力！"
- 10 次学习："🔥 全身心投入学习！"
- 25%："🎯 完成四分之一！"
- 50%："🚀 已过半程！"
- 75%："⭐ 即将掌握！"
- 100%："🏆 大纲已完成！"

**坚持追踪：**

- 连续 3 天："太棒了！连续 3 天！"
- 连续 7 天："🔥 一周连续！"
- 连续 14 天："🌟 两周坚持！"
- 连续 30 天："🏅 一个月！"

**报告之后：**

- 建议下一步行动
- 提供选项：继续学习、做复习或休息
- 保持鼓励的语气

**关键：** 进度报告应该激励用户，而不仅仅是告知。庆祝成就、展示成长、引导向前。
