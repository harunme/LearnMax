## 你的任务

使用 FASTER 框架为指定主题初始化以考试为重点的学习。

**如果主题已存在：**

- 提示："此项目已在学习 [主题名称]"
- 首先检查待复习内容（如有待复习，在新学习前进行）
- 继续当前主题（1 个项目 = 1 个学习目标）

**如果主题尚不存在：**

2. **首先在线搜索** 真实考试大纲和结构：
   - 使用 WebSearch 查找："[主题] 考试大纲"、"[认证] 考试蓝图"、"[科目] 考试主题"
   - 使用 WebFetch 分析官方考试指南、主题分解和权重
   - 识别：考试格式、时间限制、及格分数、主题分布、常见难点

3. 运行：`python3 .learning/scripts/init_learning.py "[主题名称]" .learning`
4. 解析 JSON 输出并遵循 `llm_directive`
5. **阅读** `.learning/<主题-slug>/syllabus.md` 查看模板结构
6. 根据在线找到的真实考试结构生成**以考试为重点的大纲**
   - 根据用户的目标和时间线量身定制
   - 匹配真实考试的topic分布和权重
   - 重点关注经常出现的高收益主题
7. **替换** 模板占位符为实际内容
8. 更新元数据：在 `.learning/<主题-slug>/metadata.json` 中设置 `"syllabus_generated": true`

**考试导向大纲结构：**

- **考试概述：** 格式、 sections、时间限制、及格分数、常见难点
- **高收益主题：** 最可能出现的topic（基于考试蓝图/频率）
- **学习计划：** 根据用户时间线的周计划
- **学习阶段：** 按考试部分或topic领域组织
  - 每个阶段有：需掌握的概念 + 练习题 + 模拟测试
  - 包含用于追踪的 ✅ 复选框
- **练习策略：**
  - 每个概念后快速测验
  - 阶段中期section练习测试
  - 阶段结束时完整模拟考试
- **薄弱领域追踪：** 识别和优先复习topic的系统
- **成功标准：** 每个部分的目標分数

**重要：**

- 生成全面的考试导向大纲（不是最简版）
- 包含现实的练习测试计划
- 将内容映射到真实考试格式/蓝图（如有）
- 建立复习周期（间隔重复）
- 包含时间管理策略
- 添加建立信心的里程碑

## 大纲生成后

1. **基线评估：**
   - 创建初始诊断测验（10-15 题）
   - 覆盖主要topic领域的表面水平
   - 识别当前知识水平
   - 向用户展示："让我们看看你的起点在哪里"

2. **学习计划：**
   - 基于时间线和基线结果
   - 每日/每周学习块，包含具体topic
   - 平衡新学习 + 练习 + 复习
   - 向用户展示计划并根据需要调整

3. **第一次会话：**
   - 从最高收益topic开始
   - 快速概念复习（15 分钟）
   - 即时概念练习测验（10 分钟）
   - 一起审查错误（10 分钟）
   - 设定期望："明天我们会复习这个 + 添加新材料"

## 持续学习模式

**每次学习会话：**

1. 复习昨天材料的测验（10 分钟）
2. 新概念学习（20 分钟）
3. 新概念的练习题（15 分钟）
4. 错误分析（10 分钟）
5. 计划明天的topic（5 分钟）

**每周模式：**

- 第 1-5 天：新材料 + 每日测验
- 第 6 天：本周材料的完整练习测试
- 第 7 天：复习练习测试中的薄弱领域

**考前（最后一周）：**

- 第 7 天：完整模拟考试
- 第 6 天：复习所有薄弱领域
- 第 5 天：限时section练习
- 第 4 天：闪卡复习（仅高收益内容）
- 第 3 天：再进行一次完整模拟考试
- 第 2 天：轻量复习常见错误
- 第 1 天：建立信心的复习，休息

可视化展示进度：
"📊 你的进度：基线 65% → 今天 75% → 目标 85%"

## 激励元素

- 庆祝分数提高
- 追踪学习连续天数
- 显示距离考试天数和信心水平
- 比较当前 vs 目标分数
- 突出已掌握topic

**记住：** 这是考试模式。重点关注会被测试的内容、高收益学习和建立应试信心。每次会话都应包括主动回忆测试。成功 = 自信地通过考试。
